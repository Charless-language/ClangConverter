# オペコード対応状況レポート

`Charless-cli/command-set-v3-design.md` (v3) に対するコンバータの対応状況調査結果です。

## 未対応 / 不足しているオペコード

### `charless2c.py` (Decompiler)
以下のオペコードが実装されていませんが、仕様には存在します。特に **太字** のものは `c2charless.py` が生成するため、デコンパイル不能になるリスクがあります。

- **804 (GTE): `>=`**
- **805 (LTE): `<=`**
- 800 (EQS): 文字列比較
- 030 (MEM_STR_OUT): メモリ中文字列出力
- 102 (STR_INPUT): 文字列入力

### `c2charless.py` (Compiler)
コンパイラはC言語のサブセットを扱うため全命令を使うわけではありませんが、現状以下のv3命令は生成されません（これは仕様上の制限として許容範囲かもしれません）。

- 030 (MEM_STR_OUT)
- 102 (STR_INPUT)
- 800 (EQS) (文字列比較は未サポート)
- 100 (INPUT_CHAR) (コード生成ロジックには定数定義があるが `scanf` 以外で生成されないかも？ -> `getchar` も未サポート？)

## 推奨アクション
1. `charless2c.py` に `804 (GTE)` と `805 (LTE)` のサポートを追加する（緊急性が高い）。
2. 残りのオペコードは必要に応じて実装する。
